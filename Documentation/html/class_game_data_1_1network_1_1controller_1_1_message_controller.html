Detailed Description

Base class of all message controllers

The message controller implement the minimal functionality of any message controller. In this case, every message controller must deal with debug messages.
In general the purpose of a message controller is to create the messages from the "context" and provide callbacks, that execute code and interact with the "context", if a certain message is received. So the message controller is kind of the interface from network (and outside world) with the inner "context", which is e. g. the server or the spectator client.
Therefore is has do<Message>() and on<Message> methods for every message type, the context need to implement. The do-Methods always get information and create a message from this information and tell the network controller to send it and the on-Methods are always triggered by the network controller (respectively connection handler) and parse as well as process a message and so effect the "context".
